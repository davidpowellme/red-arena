Buttons and Links

There should be an abundance of clickable content throughout the app. There are certain styles to follow to ensure that these elements can be easily found and utilized by the user.

Buttons

The Buttons should be used in the app whenever the user wishes to submit content which they have decided to input or make a choice between options. They should be easy to recognize and called out with colors and sizes. They should also follow uniform specifications to ensure a consistent flow throughout the site. Displayed buttons have a padding of 2em on the sides and 1em on the top and bottom. They have a width of 7em and a border radius of 2px. Buttons are not intended to be used in any headers or on the Primary Color of the site. The type should be in all caps.

Primary buttons are the main button featured in the app. They should be featured in the main content, Content Colored background, pages of the site. They should not be used on the Primary or Secondary colors. They should be used as the main action by the user on form modals and pages. For example, all "Submit" buttons should be a Primary Button. Header Font should be used for text color.

The Secondary Buttons should be used when there is a need for a button not intended to be the main action of the page or form. The button itself is transparent over the colored background with a Primary Color border of solid 2px. It can be used on the Content Color, Secondary Color, or Card Color backgrounds. The hover state of the button fills in the button with Primary Color. They should be used as a secondary, or optional, action the user can take on a form modal or page.

An alternate Secondary Button should have Font Color for its type. This is so the type is visible, since the background is transparent. On hover, the type should switch back to the Header Font used for other buttons.

In instances where the user has two buttons to decide between two actions, the Primary Button should be the button with top hierarchy. For example, if the buttons are vertical, the Primary Button should be on top. When horizontal, the Primary Button should be on the left.

Collapsable Content

There are certain times in the app where content is hidden and can be revealed via a collapsable toggle click. This allows for more clean, simple outlines for the site. It is also an aid to the user, allowing them to only view the content needing immediate attention.

For times when the collapsable content is located with the Content Color in the background, the simple line with a right chevron icon should suffice to call the users attention to it. The right chevron should also have a circular Primary color background.

Links

There should be a link whenever there is a need to link to an outside page. When on the main content pages, the link should be underlined and have a color which helps call it out, Primary Color (red).
